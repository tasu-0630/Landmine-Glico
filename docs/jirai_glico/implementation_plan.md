# 地雷グリコ 実装計画書

## 概要
「地雷グリコ」は「じゃんけんして階段を上る」ルールに「地雷」の要素を加えた2人用ローカルゲームです。本プロジェクトでは、このゲームをドット絵風のWebアプリケーションとして実装します。

## 確定した仕様 (Decisions)

1. **ゲームの進行インターフェースについて**
   1台の画面（PC等）を用い、一方が操作（地雷設置やじゃんけんの手の選択）して画面を隠し、次にもう一方が操作するターン制方式で進行します。（じゃんけんの選択がお互いに見えないように配慮したUIとします）
2. **「ドットで表現」について**
   フリーの「ピクセルアート風フォント」および、CSSを用いたドット絵風UI（レトロな四角いブロックを並べたデザイン）を使用します。
3. **効果音・テキストについて**
   「ボオン！」「ブィィィン」などの演出には、実際の音声ファイル（.mp3 や .wav など）を使用し、ブラウザ上で再生を行います。音声素材はフリー素材（もしくは生成したもの）をプロジェクトに組み込んで使用します。

## 提案する変更内容 (Proposed Changes)

### フロントエンド基盤 (Vite + React + TypeScript)
- Vite環境の構築 (React + Vanilla CSS)
- ドット絵風フォント（例: Google Fontsの「DotGothic16」やフリーのピクセルフォント等）をインポート。
- 全体的なCSS設計をレトロゲーム風（黒背景、白文字や原色のドットUI）で統一。

### コンポーネントおよび状態管理
以下のフェーズを管理するゲームループを実装します。
- **設定フェーズ**:
  - 各プレイヤーが1〜45段の中で3つの段に地雷をセットする画面。
  - 希望かぶりの判定と、かぶった場合（その段数のみを公表して）に再設定するロジック。
- **ゲーム（じゃんけん）フェーズ**:
  - 「グーリーコ」のかけ声テキスト表示と、じゃんけんボタンの表示。
  - 1ターンごとの勝敗判定。「グー」で勝ったら3段、「チョキ」「パー」で勝ったら6段進むロジック。
  - **あいこ5回連続の特殊処理**: 同じ手によるあいこが5回連続した場合、現在位置（同じ段の場合は到着ターンの早さ）を比較して勝者を決定し、勝者が3段か6段かを選択するUIの表示。
- **移動・地雷判定フェーズ**:
  - 階段の視覚的UIと、プレイヤーの現在位置の表示。
  - 移動後の地雷踏み判定。相手の地雷なら「被弾」として爆発演出・10段後退、自分の地雷なら「ミス」としてブザー演出・ペナルティなし。
  - ペナルティ処理は該当プレイヤーのみに行う。地雷は踏んでも残り続ける仕様。
- **結果フェーズ**:
  - 勝者（46段に先に到達したプレイヤー）の表示とリセットボタン。

## 検証計画 (Verification Plan)
### 自動/手動テスト
ゲームの性質上、ブラウザでの手動操作によって要件を満たしているか動作確認を行います。
1. **地雷設置テスト**: 0段目、46段目に設置できないこと。相手との希望がかぶった場合に正しく再設定になることの確認。
2. **じゃんけん・移動テスト**: 各手に応じた歩数（グー3, チョキ6, パー6）が正しく加算されることの確認。
3. **地雷発動テスト**:
   - 自分の地雷を踏んだ際に10段下がらずミス扱いになること。
   - 相手の地雷を踏んだ際に10段下がるペナルティを受けること。
   - 何度踏んでも地雷が消えずに残り続けること。
4. **特殊勝利テスト**: 同じ手で5回あいこになった時、ルール通りにより上の段にいる（あるいは早く着いていた）プレイヤーが勝利し、段数選択ダイアログが出るかの確認。
